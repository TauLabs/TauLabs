00032 /*
00033  * Description of DeviceLink Protocol:
00034  * A Request is initiated with R/ followed by id's of to be requested settings
00035  * even id's indicate read only values, odd are write only
00036  * (usually id =get value id+1= set - for same setting)
00037  * id's are separated by /
00038  * requests can contain values to set, or to select a subsystem
00039  * values are separated by \
00040  * example: R/30/48/64\0/64\1/
00041  * request read only settings 30,48 and 64 with parameters 0 and 1
00042  * the answer consists of an A followed by id value pairs in the same format
00043  * example: A/30\0/48\0/64\0\22/64\1\102/
00044  *
00045  * A full protocol description as well as a list of ID's and their meanings
00046  * can be found shipped with IL2 in the file DeviceLink.txt
00047  *
00048  * id's used in this file:
00049  * 30: CAS in km/h (float)
00050  * 32: vario in m/s (float)
00051  * 38: angular speed 째/s (float) (which direction? azimuth?)
00052  * 40: barometric alt in m (float)
00053  * 42: flight course in 째 (0-360) (float)
00054  * 46: roll angle in 째 (-180 - 180) (floatniguration)
00055  * 48: pitch angle in 째 (-90 - 90) (float)
00056  * 80/81: engine power  (-1.0 (0%) - 1.0 (100%)) (float)
00057  * 84/85: aileron servo (-1.0 - 1.0) (float)
00058  * 86/87: elevator servo (-1.0 - 1.0) (float)
00059  * 88/89: rudder servo (-1.0 - 1.0) (float)
00060  *
00061  * IL2 currently offers no useful way of providing GPS data
00062  * therefore fake GPS data will be calculated using IMS
00063  *
00064  * unfortunately angular acceleration provided is very limited, too
00065  */
