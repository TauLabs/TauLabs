00051 /* The submit editor presents the commit message in a text editor and an
00052  * checkable list of modified files in a list window. The user can delete
00053  * files from the list by unchecking them or diff the selection
00054  * by doubleclicking. A list model which contains the file in a column
00055  * specified by fileNameColumn should be set using setFileModel().
00056  *
00057  * Additionally, standard creator actions  can be registered:
00058  * Undo/redo will be set up to work with the description editor.
00059  * Submit will be set up to be enabled according to checkstate.
00060  * Diff will be set up to trigger diffSelected().
00061  *
00062  * Note that the actions are connected by signals; in the rare event that there
00063  * are several instances of the SubmitEditorWidget belonging to the same
00064  * context active, the actions must be registered/unregistered in the editor
00065  * change event.
00066  * Care should be taken to ensure the widget is deleted properly when the
00067  * editor closes. */
