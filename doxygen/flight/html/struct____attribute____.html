Detailed Description

Object metadata, each object has a meta object that holds its metadata. The metadata define properties for each object and can be used by multiple modules (e.g. telemetry and logger)

The object metadata flags are packed into a single 16 bit integer. The bits in the flag field are defined as:

Bit(s) Name Meaning ------ ---- ------- 0 access Defines the access level for the local transactions (readonly=1 and readwrite=0) 1 gcsAccess Defines the access level for the local GCS transactions (readonly=1 and readwrite=0), not used in the flight s/w 2 telemetryAcked Defines if an ack is required for the transactions of this object (1:acked, 0:not acked) 3 gcsTelemetryAcked Defines if an ack is required for the transactions of this object (1:acked, 0:not acked) 4-5 telemetryUpdateMode Update mode used by the telemetry module (UAVObjUpdateMode) 6-7 gcsTelemetryUpdateMode Update mode used by the GCS (UAVObjUpdateMode)
